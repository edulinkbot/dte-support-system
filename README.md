# DTE Support System

เว็บแอป prototype สำหรับสาขาวิชาเทคโนโลยีดิจิทัลเพื่อการศึกษา คณะครุศาสตร์ มหาวิทยาลัยราชภัฏบ้านสมเด็จเจ้าพระยา ใช้เป็นระบบสนับสนุนบริการนักศึกษา งานสาขา และหลักฐานเชิงระบบสำหรับ SAR/AUN-QA

## ระบบนี้คืออะไร

DTE Support System เป็นเว็บศูนย์กลางข้อมูลหลักสูตร บริการนักศึกษา การให้คำปรึกษา การติดตามนักศึกษาเสี่ยง คำร้องทางวิชาการ ข้อร้องเรียนแบบรักษาความลับ สิ่งสนับสนุนการเรียนรู้ แบบประเมิน และ dashboard งานประกันคุณภาพ

## ใช้เพื่ออะไร

ใช้สาธิต workflow และหน้าจอหลักที่สามารถเชื่อม URL จากเว็บสาขาได้ และใช้จับภาพหน้าจอเป็นหลักฐานประกอบ SAR/AUN-QA โดยเฉพาะ C4, C6, C7 และ C8

## โครงสร้างระบบ

- `src/App.jsx` ควบคุมหน้าและ local state
- `src/data/mockData.js` ข้อมูลจำลองของหลักสูตร รายวิชา คำร้อง feedback การให้คำปรึกษา และ QA
- `src/components` component กลาง เช่น Header, Navigation, Card, Table, Form, Flow
- `src/pages` หน้าหลัก 12 หน้า
- `src/styles/index.css` Tailwind และ style เสริม

## ข้อมูลจำลอง

ระบบมี mock data อย่างน้อย 20 รายการ แบ่งเป็นคำร้องทั่วไป 6 รายการ ข้อร้องเรียนลับ 4 รายการ feedback 4 รายการ ข้อมูลการให้คำปรึกษา 3 รายการ และข้อมูลนักศึกษาเสี่ยง 3 รายการ โดยใช้ชื่อจำลองทั้งหมด เช่น นักศึกษา ก นักศึกษา ข นักศึกษา ค

เพิ่มข้อมูลนักศึกษาจำลองรายชั้นปีสำหรับใช้ทดสอบระบบทะเบียนและการติดตาม ได้แก่ ปี 1 ประจำปีการศึกษา 2569 จำนวน 21 คน รหัสขึ้นต้น 69, ปี 2 รหัสขึ้นต้น 68, ปี 3 รหัสขึ้นต้น 67 และปี 4 รหัสขึ้นต้น 66 โดยไม่ใช้ชื่อบุคคลจริงจากเอกสารหรือภาพหน้าจอ

เพิ่มข้อมูลอาจารย์ประจำหลักสูตรโดยอ้างอิงรายชื่อและคุณวุฒิจากเล่ม มคอ.2/หลักสูตรในโฟลเดอร์ DTE67 พร้อมรูปภาพประกอบใน `public/faculty` และเพิ่มข้อมูลรายวิชาที่เกี่ยวข้องกับหลักสูตรปรับปรุง พ.ศ. 2567 และ พ.ศ. 2562

## ความสัมพันธ์กับ AUN-QA

- C4: ข้อมูลหลักสูตร PLO โครงสร้างหลักสูตร รายวิชา และ CLO/PLO mapping
- C6: ระบบอาจารย์ที่ปรึกษา บริการนักศึกษา feedback และการติดตามนักศึกษาเสี่ยง
- C7: สิ่งสนับสนุนการเรียนรู้ ห้องปฏิบัติการ ระบบ MIS e-learning และสถานะทรัพยากร
- C8: dashboard, improvement record, feedback loop และข้อมูลที่นำไปปรับปรุงหลักสูตร

## ข้อจำกัดของ prototype

- ใช้ mock data และ local state เท่านั้น
- ยังไม่มี login จริง
- ยังไม่เชื่อมฐานข้อมูล
- ยังไม่มี notification จริง
- ยังไม่ export report จริง
- ยังไม่เชื่อมกับเว็บสาขาหรือระบบมหาวิทยาลัยจริง
